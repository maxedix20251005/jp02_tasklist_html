# メモまとめ（日本語・詳細版）

## 1. アプリ概要
- 純粋な HTML/CSS/JavaScript のタスクリストアプリ。
- バックエンドなし。データはブラウザ内に保存。
- 保存場所: C:\academia\src\jp02_tasklist_html

## 2. 保存方式の流れ
- 最初は localStorage を採用（簡単のため）。
- file:// + module スクリプトの組み合わせで確認画面の問題が発生。
- file:// 制限を避けるため、通常の script タグに変更。
- 永続性と拡張性のため IndexedDB に移行。

## 3. IndexedDB の永続性
- ブラウザを閉じてもデータは残る。
- データはブラウザのプロファイル配下にディスク保存（HDD/SSD）。

## 4. MySQL がバックエンド必須な理由
- ブラウザは MySQL/PostgreSQL に直接接続できない。
- 必ずバックエンド/API またはサーバレスが必要。
- フロントのみの選択肢は localStorage / IndexedDB / File System Access API。

## 5. Chrome の IndexedDB 保存場所
既定プロファイルのパス:
C:\Users\<your-username>\AppData\Local\Google\Chrome\User Data\Default\IndexedDB\

フォルダ命名規則:
- https サイト: https_<host>_0.indexeddb.leveldb
- http サイト: http_<host>_0.indexeddb.leveldb
- ローカルファイル: file__0.indexeddb.leveldb（file:// はまとめて 1 つ）

## 6. 対象フォルダの特定方法
1. アプリを開き、タスクを追加/編集する。
2. Explorer で IndexedDB フォルダ一覧を更新。
3. 更新日時が最新のフォルダが対象。

## 7. DevTools からの確認（Chrome）
- F12 -> Application タブ -> IndexedDB
- Database: tasklist_db
- Stores: todos, categories, state

## 8. file:// のスクリプト注意点
- file:// では module スクリプトが制限される場合がある。
- 本アプリは通常 script で動作。

## 9. 初期データの扱い
- IndexedDB が空の場合のみ初期データを投入。
- データがある場合は既存データを使用。

## 11. 現在のバリデーション UX
- 作成/編集フォームで必須、数値範囲、日付前後関係を検証する。
- Due Date は Start Date 以降である必要がある。
- エラー項目は赤系の強調表示になる。
- 送信時は最初のエラー項目へフォーカス移動する。

## 12. 現在の補助ガイダンス UX
- 作成/編集フォームで Start Date、Due Date、% Complete に補助テキストを表示する。
- Task Name と Description はリアルタイムで文字数表示する。
- 文字数ガイダンスは入力中に更新される。

## 13. 現在の確認画面 UX
- Confirm 画面では Basic Information、Schedule、Progress に分けて表示する。
- Save 前の視認性と確認しやすさを高める目的。

## 14. 現在の空状態 / 成功状態 UX
- 一覧画面では、絞り込み結果が 0 件のときに案内付きの空状態を表示する。
- 空状態では Reset Filters と Create Task を案内する。
- 完了画面では保存内容の上に成功バナーを表示する。

## 15. 現在のテーブル UX
- 一覧テーブルは長い結果でも見やすいようにヘッダー固定にしている。
- ソート方向はより分かりやすい表示にしている。
- 長いタスク名や担当者名はレイアウト崩れを防ぐため抑制表示している。
- 一覧の列見出しは横幅を抑えるため短縮表示している。
- Delete は軽い破壊的操作表示にし、Edit を主操作として残している。
- RAG は文字なしの小さな円形カラー表示にしている。
- ページングは 1 ページ 10 件固定で、`前へ`、`次へ`、ページ番号、総件数/表示範囲を表示する。
- Due Date は、期限切れを赤、3日以内を黄で強調表示する。
- 一括削除は、各行チェック、表示中の全選択、確認ダイアログ付きで対応している。

## 16. 現在のアクセシビリティ UX
- 主な操作部品でキーボードフォーカスを見やすくしている。
- フォーム項目は補助文言とエラー文言に関連付けている。
- バリデーションエラーは専用のエラー領域で通知する。
