How to Use the Task List

Getting Started

Open index.html in your browser to start using the task list. All data is stored locally in your browser using IndexedDB.

Viewing Tasks

  • The main page shows all tasks in a table format.
  • Use the search box to filter tasks by name.
  • Use the category dropdown to filter by category.
  • Click column headers to sort the list.
  • On screens 600px or narrower, the view switches to cards for better mobile usability.
  • Overdue tasks are highlighted in red; tasks due within 3 days are in yellow.
  • Click a task name to view details.

Creating and Editing Tasks

  • Click "New Task" to create a new task.
  • Fill in the required fields: Task Name and Assigned To.
  • Optional: Description, Start Date, Due Date, % Complete, Category.
  • Due Date must be on or after Start Date.
  • % Complete should be between 0-100.
  • Click "Confirm" to review, then "Save" to store.
  • To edit, click "Edit" from the actions column or detail view.

Other Features

  • Pagination: 10 items per page with navigation.
  • CSV Export: Download the current filtered list as CSV.
  • Bulk Delete: Select rows with checkboxes and delete multiple tasks.
  • Reset: Clear all filters to show all tasks.

Troubleshooting

If you encounter issues, ensure JavaScript is enabled and you're using a modern browser (Chrome, Firefox, Edge).

Technical Details

Architecture

This application is a single-page, frontend-only task management system built with pure HTML, CSS, and JavaScript.

Data Storage

  • Technology: IndexedDB
  • Persistence: Data persists across browser sessions on the user's device
  • Database: tasklist_db
  • Stores: todos, categories, state
  • No Server: All processing and storage happen locally in the browser

Features

  • List, search, filter, and sort tasks
  • Create, read, update, and delete (CRUD) operations
  • Fixed pagination (10 items per page)
  • CSV export of filtered results
  • Bulk operations (delete selected)
  • Responsive design with mobile card view (≤ 600px)
  • Form validation and confirmation workflow

Responsive Design

  • Desktop: Table view with all columns and features
  • Tablet (900px): Toolbar wraps; non-critical columns hidden
  • Mobile (≤ 600px): Card view for better touch interaction; input controls shrink to prevent horizontal overflow

Browser Support

Modern browsers with JavaScript enabled and IndexedDB support (Chrome, Firefox, Safari, Edge)

Files

  • Frontend: assets/app.js, assets/styles.css
  • Pages: index.html (list), create.html (form), confirm.html (review), complete.html (success), view.html (detail)
  • No Backend: This is a client-side application with no server dependencies
